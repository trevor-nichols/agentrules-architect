# EP-20260602-002 - Add Anthropic Opus 4.7 and 4.8 support

This ExecPlan is a living document. Keep `Progress`, `Surprises & Discoveries`, `Decision Log`, and `Outcomes & Retrospective` up to date as work proceeds.

If `.agent/PLANS.md` exists in this repository, maintain this plan in accordance with that guidance.

## Purpose / Big Picture

Users can select `claude-opus-4.7` and `claude-opus-4.8` from the built-in model preset registry without tripping invalid Anthropic Messages API payloads. The direct Anthropic provider and the Claude Code runtime will both emit adaptive thinking for Opus 4.7 and 4.8, accept the new `xhigh` effort level where the model family supports it, and keep older Claude families on their existing request shapes. End-to-end verification comes from the targeted Anthropic and Claude Code unit tests plus preset-registry tests.

## Scope

In scope: add new Claude Opus 4.7 and 4.8 model configs and presets; update Anthropic capability metadata; fix request builders so Opus 4.7 and 4.8 no longer send `thinking: {type: "enabled", budget_tokens: ...}`; add `xhigh` effort support where Anthropic documents it; update Claude Code derived presets for the same model families; extend unit tests to cover the new payload rules.

Out of scope: implement Anthropic beta-only features such as fast mode, task budgets, or summarized thinking display; change prompt logic; change offline fixtures; add new CLI flows.

## Progress

- [x] Initial draft created.
- [x] Added Anthropic model configs and preset registry entries for Claude Opus 4.7 and 4.8.
- [x] Updated Anthropic and Claude Code request builders to respect adaptive-only Opus 4.7/4.8 thinking and `xhigh` effort support.
- [x] Validation complete.

## Surprises & Discoveries

- Anthropic's latest effort docs also expand `claude-sonnet-4-6` to support `max`, so the existing capability matrix was already stale before adding Opus 4.7 and 4.8.
- Claude Code request building had the same adaptive-thinking blind spot as the direct Anthropic Messages path, so adding runtime presets for Opus 4.7 and 4.8 without fixing the builder would have produced invalid or degraded requests.
- The repository's direct Anthropic path and Claude Code path shared the same conceptual bug but expressed it differently: the Anthropic builder validated `output_config.effort`, while Claude Code silently dropped unsupported effort values. Bringing both builders onto the same capability metadata eliminated that drift.

## Decision Log

- Keep fast mode and task budgets out of scope for this change. They require beta-surface request handling and are not necessary to make Opus 4.7 and 4.8 selectable and valid in the default provider path.
- Treat `ReasoningMode.ENABLED` as "thinking on" for Opus 4.7 and 4.8 by translating it to adaptive thinking, because those families reject manual `budget_tokens` mode outright. Older families that still accept manual thinking keep the existing payload shape.
- Add Claude Code runtime presets for Opus 4.7 and 4.8 in the same change, because the repository exposes Claude Code as a first-class provider and those presets would otherwise lag behind the direct Anthropic catalog.

## Outcomes & Retrospective

The repository now exposes Claude Opus 4.7 and Claude Opus 4.8 as first-class Anthropic presets, plus matching Claude Code runtime presets. Anthropic effort typing now includes `xhigh`, capability metadata recognizes Opus 4.7 and 4.8 as adaptive-only families, and both request builders translate reasoning modes into payloads that match Anthropic's current API contract.
